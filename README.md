# はじめに
このリポジトリは指定された情報に基づいてテック記事(Qiitaフォーマット)を作成するためのプロンプトになります。

# 利用手順
## 前提条件
このプロンプトはVS CodeからGitHub Copilotを使ってAIに記事を書いてもらうことを想定しています。<br>
それ以外の環境では想定した挙動にならない可能性があるのでご注意ください。<br>
<br>
また、現時点では著作権対応していないため、作られた記事は自分でも目視チェックを**必ず**行ってください。<br>

## 1. このリポジトリをクローンする
## 2. 以下のプロンプトにテック情報（テキスト or URL）を与えて実行する
```
/01_記事整理
```

## 3. 以下のプロンプトを実行する
```
/02_記事セクション作成
```

## 4. 以下のプロンプトに文字数を与えて実行する
```
/03_記事作成
```

## 5. 以下のプロンプトを実行する
```
/04_記事レビュー
```

# 文体・トーンの作り方

現在、文体・トーンを抽出するためのカスタムプロンプトを作成予定です。それまでの間は以下の対応で「文体・トーン」を抽出可能です。
## 1. Chat-GPT(無料プランでOK)に以下のように、文体・トーンを抽出するためのURLを貼り付けてお願いします。
```
以下URLからAIに記事を書いてもらうために「ルール」として使用するための「文体・トーン」をmarkdownで抽出してください
https://qiita.com/ochtum/items/750413e44b0d296387ef
```

## 2. 抽出された「文体・トーン」を`.specify\memory\文体・トーン-constitution.md`に貼り付けます。

# フォーマット変更
Qiita以外のフォーマットにしたい場合、`templates\記事作成-template.md`を変更することでフォーマット変更できます。

# 記事のデザインについて
`.specify\memory\文体・トーン-constitution.md`の「セクションデザインのルール」～「手順・注意点・Tips の明示」のセクションに記載されています。<br>
こちらを変更して頂くことで、記事のデザインを変更して頂くことが可能です。
